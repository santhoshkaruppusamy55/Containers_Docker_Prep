
## Container vs Virtual Machine

> Both are used to **isolate applications & their dependencies**.

| Feature | Containers *(small in size)* | VMs *(large in size)* |
|---------|------------------------------|------------------------|
| **Resource Utilization** | Shares host OS kernel → lighter & faster *(dynamic resource)* | Full-fledged OS + hypervisor *(static resource)* |
| **Portability** | Portable — runs on any system with compatible host OS | Less portable — needs compatible hypervisor |
| **Security** | Less isolation — shares host OS | High security — each VM has separate OS, isolated from host & other VMs |
| **Management** | Easy | Difficult |
