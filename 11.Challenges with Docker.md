## Real-Time Challenges with Docker

| # | Challenge | Details |
|---|-----------|---------|
| **1** | Single point of failure | Docker is a **single daemon process** — if daemon goes down, all applications go down for some reasons, all applications are down *(Use **Podman** — open-source, daemonless, rootless)* |
| **2** | Security — root user | Docker daemon runs as **root user** — any compromised process can impact other containers or host. |
| **3** | Resource constraints | Running too many containers on a single host → slow performance or crashes,Not just docker, but containers themselves too. |
