
## Files & Folders in Container Base Images

| Directory | Purpose |
|-----------|---------|
| `/bin` | Binary executable files — `ls`, `cp` |
| `/sbin` | System binary files — `init`, `fdisk`, `ifconfig` *(administration)* |
| `/etc` | Configuration files for system services |
| `/lib` | Library files for `/bin` & `/sbin` |
| `/usr` | User-related files & utilities |
| `/var` | Variable data, log files |
| `/root` | Home directory of root user |

> These help make containers **isolated from each other**.
> **Linux Kernel Scheduler** ⇒ Manages process execution

## Files & Folders Containers Use from Host OS

| Resource | Description |
|----------|-------------|
| **i) Host's File System** | Docker containers use bind mounts to read & write files in host file system |
| **ii) Networking Stack** | Provides network connectivity to the container |
| **iii) System Calls** | Host kernel handles system calls from container — how containers access memory & I/O |
| **iv) Namespaces** | Isolate system resources / processes |
| **v) Control Groups (cgroups)** | Control CPU & RAM |

### iv) Namespaces — Details

- Docker containers use Linux namespaces to create **isolated environments** for container processes
- Namespaces provide isolation for: file system, process ID & network

| Namespace | Purpose |
|-----------|---------|
| `PID` | Process ID isolation |
| `NET` | Network isolation |
| `MNT` | Mount/filesystem isolation |
| `IPC` | Inter-process communication |
| `UTS` | Hostname isolation |
| `USER` | User ID isolation |

### v) Control Groups (cgroups) — Details

- Docker containers use cgroups to **limit & control** the amount of resources a container can access
- Helps monitor resource usage to prevent one container from consuming all host resources
