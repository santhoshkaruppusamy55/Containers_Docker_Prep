
## Docker Volumes & Bind Mounts

> `-v path` flag while running the image

- Container is **ephemeral** (short-lived) — if it gets killed, log files are lost
- When a container is deleted, its **writable layer is removed** → log files & data are lost

| Type | Use Case |
|------|----------|
| **Docker Volumes** | Production |
| **Bind Mounts** | Development |

```js
// Example in project
fs.appendFileSync("app.log", "user logged in")   // → /app/logs  → will be gone when container dies
```

### Docker Volumes

> Usually stored in `/var/lib/docker/volumes/`

- Docker creates & manages a specific folder in the host
- If path doesn't exist → creates automatically

```bash
docker run -v mylogs:/app/logs my-app
```

### Bind Mounts

- Data is stored in a **specific directory you choose**
- If path doesn't exist → creates automatically

```bash
docker run -v /home/santh/logs:/app/logs my-app  # Docker volume
# OR
docker run --mount type=bind,source=/home/santh/logs,target=/app/logs my-app # Bind Mounts
```

### Volume Commands

```bash
docker volume create volume_name     # Create a volume
docker volume rm volume_name         # Delete a volume
```
