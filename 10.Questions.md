
## Interview Questions

**1. What is Docker?**
> Open-source containerization platform to build, run & push container images.

**2. VM vs Container?**
> See Container vs Virtual Machine file

**3. What is Docker lifecycle?**
```
build → push → pull → run
Dockerfile → Image → Container
```

**4. Different Docker components?**
> `Docker Daemon`, `Docker Registry`, `Docker Image`, `Docker CLI`, `Docker Desktop`, `Containers`

**5. Difference between COPY and ADD in Docker?**

| `COPY` | `ADD` |
|--------|-------|
| Can only copy files from host system into container | Can copy files from a URL also |

**6. Difference between ENTRYPOINT & CMD in Docker?**
> See DockerFile.

**7. What are networking types in Docker & what is created by default?**

| Type | What is Created |
|------|----------------|
| **Bridge** *(default)* | `veth`, `eth0`, `bridge`, `docker0` are created |
| **Host** |  it bind our conatiner networks with host OS - access applications uisng Host Ip |

**8. How to isolate networking between containers?**
> By default, using `veth` & `docker0`, containers connect to Host OS.
> To isolate a specific container → create your own bridge network & run the container with that custom network.

**9. How do containers communicate with each other?**
> Connect all containers to the **same network** to make them communicate with each other.

**10. What is multi-stage build in Docker?**
> - To build **lightweight** containers
> - Multiple `FROM` commands in a single Dockerfile
> - Only runtime dependencies (no build tools) in final image

**11. What are distroless images in Docker?**
> - Images contain only the application & its runtime dependencies with minimum OS libraries
> - No package managers (yum, apt), no shell, no most user tools
> - ⚠️ If you need to debug or need shell access → don't use distroless

| Standard Image | Distroless Image |
|----------------|-----------------|
| `FROM python:3.11` | `FROM gcr.io/distroless/python3` |

**12. How do containers communicate with Host OS?**
> - i) System calls
> - ii) Namespaces
> - iii) Cgroups
> - iv) Network stack
>
> *(Not by Linux distribution)*
