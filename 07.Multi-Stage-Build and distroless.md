
## Multi-Stage Build in Docker

> Uses **multiple `FROM` statements** in a single Dockerfile.

- Commonly used to **reduce Docker image size** by separating build & runtime environments
- Most useful for: **Go**, **Java (Maven build)**, **React (npm build)**

### Image Size Comparison (React + Node.js Project)

### Distroless Images

> **Distroless** ⇒ Distribution-less image — *For production containers*

A Docker image that does **not** contain a Linux distribution:

| ❌ Does NOT have | ✅ DOES have |
|-----------------|-------------|
| Package manager (apt, yum) | Required runtime libraries |
| Shell | Dynamic linker |
| Core utilities | Necessary shared libraries |
| Most Linux user tools | Our application |

**Reduce Image Size by:**
- **Multi-stage build** ⇒ Removes build tools
- **Distroless** ⇒ Improves security & further reduces size by removing Linux distribution layer & unnecessary utilities
